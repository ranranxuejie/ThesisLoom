# 用户特殊要求 (User Requirements)

1. **结构深度要求**：在撰写论文时，必须深入详述系统的体系结构（基于StateGraph），深入探讨多智能体设计（Planner、Editor、Reviewer、Revise）是如何模拟人类在撰写长篇学术著作时的分工和迭代过程的。
2. **实验与评估指标**：需要阐述该框架相比于直接 zero-shot 或 few-shot Prompting 的优势，最好加入通过评估生成的上下文连贯性、错误率（幻觉）、结构完备性和风格契合度等维度的结果。
3. **语言转换自如**：系统设计在“English”和“中文”间流畅切换，要求在论文中讨论LLM基座在不同语言语境下对“高阶专业学术词汇”的处理能力和风格对齐度。
4. **技术细节与可复现性**：展示代码逻辑层面，如何通过 state.py 中的字典传递，避免Token浪费，以及 reviewer 节点产生的二进制指标如何有效熔断不良生成循环的。
5. **通篇基调控制**：确保全文高度契合"Nature/Science 子刊"的标准，没有任何废话或凑字数行为。
