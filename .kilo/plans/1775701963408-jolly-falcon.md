# Plan: Tauri + React UI Refactoring for ThesisLoom

## Goal
Migrate the existing Streamlit-based UI (`streamlit_app.py`, `state_dashboard.py`) to a modern Tauri + React architecture, while keeping the core Python backend logic intact.

## Current Architecture
- Backend: Python (`workflow.py`, `main.py`, `api.py`, `core/*`)
- Frontend/UI: Streamlit (`streamlit_app.py`, `state_dashboard.py`)
- The UI communicates with the backend mostly via file system state (`completed_history/*_checkpoint.json`, `inputs/inputs.json`) and some shared logic.

## Proposed Architecture
- **Backend (Python)**: Serve as a sidecar to Tauri or expose a local REST/WebSocket API using FastAPI (since `api.py` already imports `fastapi`). 
- **Frontend (Tauri + React)**: A React SPA that communicates with the Python backend to read states, submit actions, and monitor workflow progress.

## Steps
1. **Initialize Tauri Project**: 
   - Create a `src-tauri` directory for the Tauri rust app.
   - Initialize a React project (e.g. using Vite) in the root or a specific `ui` folder.
   - Configure Tauri to use the React app as the frontend.

2. **Backend Adaptation**:
   - Instead of Streamlit running in a subprocess (`main.py`), we will need to adjust `main.py` to either start the FastAPI server or let Tauri manage the Python process as a sidecar.
   - Refactor the UI logic from `state_dashboard.py` and `streamlit_app.py` into API endpoints (e.g. `GET /state`, `POST /action`, `GET /projects`).

3. **Frontend Development (React)**:
   - Create components for the Dashboard:
     - Project Selector
     - Workflow State Monitor (Progress, Current Node)
     - Action Panel (Approve/Reject, Inputs)
   - Use TailwindCSS or a component library (MUI, Shadcn) for styling.

4. **Integration & Testing**:
   - Connect React frontend to the Python backend endpoints.
   - Test workflow progression through the new UI.
   - Ensure the `--ui-mode` logic in `main.py` is appropriately transitioned to launching the Tauri app.

5. **Packaging**:
   - Update build scripts (`scripts/build_exe.ps1`) to build the Tauri app instead of Streamlit.

## Questions Answered
- **Integration**: Python backend will be bundled as a Tauri sidecar (compiled with PyInstaller).
- **UI Library**: We will use Shadcn UI (with Tailwind CSS) for a modern, clean, and accessible design.
- **Legacy Support**: We will completely replace and remove the Streamlit UI.
